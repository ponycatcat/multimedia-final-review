Q1 & Q4 共同觀念 · 為什麼要做 Run-Length 編碼?

當資料是「一長串 0、偶爾才一個 1」(例如二值影像/傳真,或量化後的 DCT 係數大多是 0),直接存每個 bit 很浪費。Run-Length 的想法:只記「兩個 1 之間夾了幾個 0」,再把這個「run 長度」用一張對照表換成碼字。

💡 整題最關鍵、也是你扣分主因的觀念:題目的 run 數字 36 / 16 / 30 / 55 是每一段的「總位元數」(含結尾那個 1),不是 0 的個數。
證據:36+16+30+55 = 137 剛好等於題目說的「137-bit stream」。
所以每段「0 的個數」要 減掉那個 1:

段該段總長(題目給的 run)結尾的 1真正的 0 個數 L
136−135
216−115
330−129
455−154
0 的總數 = 35+15+29+54 = 133,加上 4 個 1 = 137 ✓L = 35,15,29,54

✗ 你的錯誤(差一錯誤 off-by-one)

你直接把 36 / 16 / 30 / 55 當成 0 的個數去編碼(卷子上寫 36=15+15+6…)。少減了結尾的 1,整段編碼就跟著錯。正確的 L 是 35 / 15 / 29 / 54。

Q2 · 這組碼是否正確?(前綴碼 Prefix Code)

給定:A=1　B=01　C=101　D=011。問正確嗎?為什麼?

前綴碼規則(Prefix-free / 可唯一解碼):任何一個碼字都不能是另一個碼字的開頭。否則解碼端讀 bit 流時會分不清字元在哪裡結束。

逐對檢查(這就是滿分寫法)

比較是否前綴?結果
A=1 vs C=101C 以 1 開頭A 是 C 的前綴 ✗
B=01 vs D=011D 以 01 開頭B 是 D 的前綴 ✗
其餘配對—OK

✗ 答案:這組碼不正確

共有兩處違反前綴規則:
① A=1 是 C=101 的前綴;② B=01 是 D=011 的前綴。
例如收到 011…:可能是「B(01)+下一字以 1 開頭」也可能是「D(011)」→ 無法唯一解碼。
老師批改聚焦在 B⊂D(「電腦無法辨視 B 與 D」),但完整作答把 A⊂C 也寫出來更穩。

✓ 怎麼改成合法前綴碼

讓每個字元都落在二元樹的葉節點。例:A=0, B=10, C=110, D=111 → 互不為前綴,可唯一解碼。
畫樹自我檢查:沿著 0/1 往下走,只有葉子能放字元,路徑中途不能停字元。

Q3 · 哪一張是高通濾波(High-pass)結果?

✓ 你答對了:選 A(這題沒扣分,但把「為什麼」講滿才拿得到解釋分)。

A = 高通 High-pass

通過高頻=影像中劇烈變化的部分=邊緣、輪廓、細節、文字筆畫;壓掉平滑區 → 平坦處變黑,只剩輪廓線條。圖 A 看得到字母與圖形的邊緣 → 高通。

B = 低通 Low-pass

通過低頻=平滑、整體亮度、緩慢漸層;壓掉高頻細節 → 影像變模糊。圖 B 朦朧模糊 → 低通。
💡 完整解釋句:「A 是高通結果,因為高通濾波保留高頻分量(邊緣/細節),衰減低頻(平滑區),所以影像只剩物體輪廓。」
這跟 DCT 的「右下=高頻=細節、左上=低頻=平滑」是同一個頻率域概念,可以一起記。

✗ 你的錯誤(這題只拿一半)

你建的是歪斜樹:把 E 直接放在樹根當葉子 → 給了 E=0(只有 1 bit)。
為什麼錯:1-bit 碼字代表「E 在樹根的其中一邊,另一邊是其他全部」。但要這樣,E 的機率必須 ≥ 另一邊(≈79%)。E 只有 21%,根本不夠 → E 不可能是 1 bit。正確下 E 應該是 2 bits(01)。
根因:你沒有「每一步重新挑兩個最小」,而是把字元一個一個往下掛成一條長鏈。只要嚴格照合併表做,就不會歪。
💡 驗算技巧:① 機率大的字元碼一定要短(E、A 最短);② 把每個碼字長度 × 機率加總,如果比別人的樹大很多,代表你的樹不是最佳。

1 · 亮度 vs 色度(Luminance vs Chrominance)

💡 白話秒懂

想像黑白電視 → 彩色電視。黑白畫面的明暗 = 亮度(Y);後來疊上去的顏色 = 色度(Cb, Cr)。
人眼是「看明暗很準、看顏色很粗」的(視網膜感光細胞:管明暗的桿細胞約 1.2 億個,管顏色的錐細胞只約 600 萬)。所以我們可以偷工減料:顏色存得粗一點(次取樣),人眼幾乎看不出來,卻能省很多資料。
亮度 Luminance (Y)色度 Chrominance (Cb, Cr)
攜帶明暗 / 灰階細節 brightness顏色資訊 color (色差 color difference)
人眼非常敏感 sensitive較不敏感 less sensitive
壓縮時保留完整解析度可次取樣 subsampling(4:2:0)

✍️ 英文作答範本

Luminance (Y) represents the brightness of an image, while chrominance (Cb, Cr) represents the color information. 亮度代表影像的明暗,色度代表顏色資訊。 Because the human eye is more sensitive to luminance than to chrominance, the chrominance components can be subsampled (e.g., 4:2:0) during compression to reduce data size with little perceptible loss of quality. 因為人眼對亮度比色度敏感,壓縮時可把色度次取樣以省資料,且幾乎看不出畫質損失。

2 · 彩色影像壓縮流程(Compression Flow of Color Images)

💡 白話秒懂

像行李打包:① 先把衣服換成好折的格式(換色彩空間)→ ② 把用不到的丟掉(色度次取樣)→ ③ 分裝成小袋(切 8×8 區塊)→ ④ 真空壓縮(DCT 轉頻率)→ ⑤ 丟掉看不出的細節(量化,就是這步會失真)→ ⑥⑦ 把空隙壓緊、綁好(RLE + Huffman,不再丟東西)。

✍️ 英文作答範本(照流程寫即可)

First, the image is converted from RGB to YCbCr. The chrominance is then subsampled. Each channel is divided into 8×8 blocks, and a DCT transforms each block into frequency coefficients. The coefficients are quantized (this is the only lossy step), then zig-zag scanned and compressed with run-length and Huffman (entropy) coding. 先 RGB→YCbCr → 色度次取樣 → 切 8×8 → DCT → 量化(唯一失真)→ zig-zag + RLE + Huffman 編碼。

3 · DCT 係數的解讀(Interpretation of DCT Coefficients)

💡 第一步:先懂「影像的頻率」

頻率 = 亮度「變化得多快」(看相鄰像素之間變化快慢):
• 平滑的天空、皮膚、牆面 → 變化很慢 → 低頻
• 斑馬條紋、頭髮、文字邊緣 → 忽明忽暗變很快 → 高頻
👉 一句話:平滑 = 低頻;細節/邊緣 = 高頻。

💡 第二步:用「音樂」理解 DCT(關鍵比喻)

一首歌可拆成低音 + 高音的混合,等化器(EQ)每根滑桿 = 「某個頻率要多大聲」。
DCT 對影像做一樣的事:把一個 8×8 方塊,拆成 64 種固定「波紋圖案」的混合,告訴你每種圖案要加多少。那個「要加多少」的數字 = 係數(coefficient)。係數大 = 這圖案很明顯;係數 ≈ 0 = 幾乎沒有。

🎨 第三步:這 64 種「波紋圖案」長這樣(程式實際畫出)

水平頻率 u(直條紋變密) ──────►
垂直頻率 v(橫條紋變密) ▼
左上角 = 一片平灰(沒波紋)= DC = 平均亮度; 右下角 = 最細棋盤格 = 最高頻(細節)。
任何 8×8 方塊,都是把這 64 張圖各乘上自己的係數再疊起來。

🔢 第四步:超小具體例子(用一排 4 個像素)

像素亮度長相係數結果能量在哪
50 50 50 50完全平[50, 0, 0, 0]只有 DC
20 40 60 80平滑漸層[50, 大, 小, ≈0]集中在低頻
80 0 80 0忽明忽暗[40, 0, 0, 大]跑到高頻

→ 越平滑,能量越集中前面(低頻);越細碎跳動,能量才跑到後面(高頻)。

💡 第五步:為什麼這樣能壓縮?

① 自然照片大多平滑 → 大數字擠左上、右下幾乎是 0(能量集中 energy compaction)。
② 人眼對高頻細節不敏感 → 右下那些小數字直接丟掉/歸零也看不太出來。
③ 一堆 0 超好壓(RLE 一次記掉)。
注意:DCT 本身不失真,它只是「把不重要的搬到右下角」,真正丟東西的是後面的量化。
英文中文
DC coefficient (top-left)直流係數 = 平均亮度
AC coefficients交流係數 = 各種頻率
low / high frequency低頻(平滑)/ 高頻(細節)
energy compaction能量集中

✍️ 英文作答範本

The DC coefficient (top-left) represents the average brightness of the 8×8 block. The other AC coefficients represent spatial frequencies: low frequencies (top-left) correspond to smooth areas, and high frequencies (bottom-right) correspond to edges and fine detail. DC=平均亮度;AC=各頻率,左上低頻(平滑)、右下高頻(細節)。 Because of energy compaction, most energy concentrates in the low-frequency coefficients, so the high-frequency ones can be discarded during quantization to achieve compression. 能量集中在低頻,高頻可在量化時丟掉以達壓縮。

4 · 卷積層(Convolution Layer)

💡 白話秒懂

想像一個小印章/放大鏡(濾波器 filter,例如 3×3),在照片上一格一格滑過去,專門找某一種圖案(例如「直線邊緣」)。哪裡有這個圖案,那裡就亮起來 → 得到一張特徵圖(feature map)。
同一個印章蓋全圖(權重共享),所以①要學的參數很少 ②圖案出現在哪都找得到(平移不變)。淺層找邊緣,深層把邊緣組成形狀、物件。

filter / kernel濾波器 / 卷積核
feature map特徵圖

✍️ 英文作答範本

A convolution layer applies small learnable filters (kernels) that slide over the input image. At each position it computes a weighted sum (dot product), producing a feature map that detects local features such as edges. 卷積層用可學習的小濾波器滑過影像,每處算加權和,產生偵測邊緣等局部特徵的特徵圖。 Thanks to weight sharing and local connectivity, it uses far fewer parameters and is translation-invariant. 因權重共享與局部連接,參數少且具平移不變性。

5 · 池化層(Pooling Layer)

💡 白話秒懂

就是縮圖:把特徵圖變小,但保留重點。看每個 2×2 小方塊,只留最大的那個值(Max pooling)或取平均(Average pooling)。
好處:資料量變小、計算變快、而且圖案稍微移位也不影響(平移不變)。重點記:池化沒有要學的權重(它只是挑數字,不像卷積有參數)。
英文中文
pooling / down-sampling池化 / 降採樣
max pooling / average pooling最大 / 平均池化
translation invariance平移不變性
no learnable parameters沒有可學習參數

✍️ 英文作答範本

A pooling layer down-samples each feature map, e.g., max pooling keeps the maximum value in each 2×2 window. It reduces computation, controls overfitting, and provides translation invariance. Pooling has no learnable parameters. 池化把特徵圖降採樣(如取 2×2 最大值),減少計算、抑制過擬合、提供平移不變性,且沒有可學習參數。

6 · CNN 訓練的兩個問題(Two Training Issues of CNN)

💡 白話秒懂

① 梯度消失:訓練時「修正訊號」要從最後一層往前傳,每經一層就乘上一個小數,層數一多就越乘越小、趨近 0,前面的層幾乎學不到東西。像一句話傳話傳到最後就聽不見了。
② 過擬合:模型把訓練照片連雜訊都背起來,考新題目(沒看過的資料)就爛掉。像學生只背考古題答案,換題就不會。

① 梯度消失 Vanishing Gradient

反向傳播時梯度連乘變極小 → 前層幾乎不更新。
解法:ReLU、權重初始化、Batch Normalization、殘差連接 (skip/residual)。

② 過擬合 Overfitting

訓練準、驗證爛(背了雜訊,無法泛化)。
解法:Dropout、資料擴增 (data augmentation)、正則化 (regularization)、Early Stopping。
英文中文
vanishing gradient梯度消失
backpropagation反向傳播
overfitting / generalize過擬合 / 泛化
dropout / regularization隨機丟棄 / 正則化

✍️ 英文作答範本

(1) Vanishing gradient: during backpropagation, gradients become very small after being multiplied through many layers, so early layers receive almost no updates and train poorly. 梯度消失:反向傳播時梯度連乘變很小,前層幾乎不更新。 (2) Overfitting: the model fits the training data too closely (even its noise) and fails to generalize to unseen data. 過擬合:過度擬合訓練資料(含雜訊),無法泛化到新資料。

7 · 三個因子的兩難(Dilemma / Trade-off of 3 Factors)

💡 白話秒懂

「魚與熊掌不可兼得」。就像「又快、又便宜、又好,只能選兩個」。數位浮水印有三個互相牽制的因子,你把其中一個拉高,另一個就會掉。
Imperceptibility 不可察覺 /\ / \ / \ Robustness ────── Capacity 強韌性 容量
  • Robustness 強韌性 — 撐得過壓縮/雜訊/裁切等攻擊。
  • Imperceptibility 不可察覺性 — 看不出來,不傷畫質。
  • Capacity 容量 — 能藏多少資料。
英文中文
trade-off / dilemma取捨 / 兩難
robustness強韌性
imperceptibility / fidelity不可察覺性 / 保真度
capacity容量

✍️ 英文作答範本

In digital watermarking there is a trade-off among three factors: robustness, imperceptibility, and capacity. Improving one usually sacrifices another — e.g., embedding more data (higher capacity) or stronger robustness tends to reduce imperceptibility. A good scheme must balance all three. 浮水印三難:強韌性、不可察覺性、容量,提高一個會犧牲另一個,須取平衡。

8 · 浮水印:LSB 與 DCT 兩種做法(LSB vs DCT Watermarking)

💡 白話秒懂

• LSB:把訊息藏在每個像素的最不重要的那一位數(最低位元)。像把價格的「個位數」偷改一塊錢——看不出來,但很容易被洗掉(一壓縮就毀)。
• DCT:先轉到頻率域,把訊息藏在中頻係數。比較麻煩,但撐得過 JPEG 壓縮(強韌)。
為何藏中頻? 低頻一改會被看見(傷畫質);高頻會被壓縮丟掉(不強韌)→ 中頻剛好兼顧。
LSBDCT

做法改像素最低位元 least significant bit改中頻係數 mid-frequency coefficients
容量 capacity高 high較低 lower
強韌性 robustness弱 weak強 strong(抗 JPEG)
複雜度簡單快 simple/fast較複雜 complex
英文中文
least significant bit (LSB)最低有效位元
spatial domain空間域
frequency / transform domain頻率 / 轉換域
embed a watermark嵌入浮水印

✍️ 英文作答範本

The LSB approach embeds the watermark in the spatial domain by replacing the least significant bit of each pixel. It has high capacity but is not robust — compression or filtering easily destroys it. LSB 在空間域改像素最低位元,容量高但不強韌,壓縮就毀。 The DCT approach embeds the watermark in the frequency domain by modifying mid-frequency coefficients. It is more robust and survives JPEG compression. (Mid-frequency is chosen because low frequencies are visible and high frequencies are removed by compression.) DCT 在頻率域改中頻係數,較強韌、抗 JPEG;選中頻是因低頻會被看見、高頻會被壓縮掉。
