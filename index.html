📝 小考錯題完整詳解

這份是你那張 69 分小考的逐題拆解。每題都先講觀念,再放完整逐步解,最後標出你卷子上實際錯在哪一步。Run-Length(Q1、Q4)是你失分最多的地方,放在最前面講最透。

Q1 & Q4 共同觀念 · 為什麼要做 Run-Length 編碼?

當資料是「一長串 0、偶爾才一個 1」(例如二值影像/傳真,或量化後的 DCT 係數大多是 0),直接存每個 bit 很浪費。Run-Length 的想法:只記「兩個 1 之間夾了幾個 0」,再把這個「run 長度」用一張對照表換成碼字。

💡 整題最關鍵、也是你扣分主因的觀念:題目的 run 數字 36 / 16 / 30 / 55 是每一段的「總位元數」(含結尾那個 1),不是 0 的個數。
證據:36+16+30+55 = 137 剛好等於題目說的「137-bit stream」。
所以每段「0 的個數」要 減掉那個 1:

段該段總長(題目給的 run)結尾的 1真正的 0 個數 L
136−135
216−115
330−129
455−154
0 的總數 = 35+15+29+54 = 133,加上 4 個 1 = 137 ✓L = 35,15,29,54

✗ 你的錯誤(差一錯誤 off-by-one)

你直接把 36 / 16 / 30 / 55 當成 0 的個數去編碼(卷子上寫 36=15+15+6…)。少減了結尾的 1,整段編碼就跟著錯。正確的 L 是 35 / 15 / 29 / 54。

✗ 你的表格錯誤

你把 L=6 的碼字填成 0111。但 0111 是十進位 7,L=6 的二進位是 0110。碼字永遠 = L 的 m 位元二進位,轉換時請逐位檢查。
編碼演算法(Q1 版):可終止的碼字最多只能表示 14 個 0。一段有 L 個 0 時 → 先寫 ⌊L÷15⌋ 個 1111(每個吃 15 個 0),剩下的 r = L mod 15(0~14)再寫成 4-bit 二進位(這一格同時代表「結尾的那個 1」)。

Q2 · 這組碼是否正確?(前綴碼 Prefix Code)

給定:A=1　B=01　C=101　D=011。問正確嗎?為什麼?

前綴碼規則(Prefix-free / 可唯一解碼):任何一個碼字都不能是另一個碼字的開頭。否則解碼端讀 bit 流時會分不清字元在哪裡結束。

逐對檢查(這就是滿分寫法)

比較是否前綴?結果
A=1 vs C=101C 以 1 開頭A 是 C 的前綴 ✗
B=01 vs D=011D 以 01 開頭B 是 D 的前綴 ✗
其餘配對—OK

✗ 答案:這組碼不正確

共有兩處違反前綴規則:
① A=1 是 C=101 的前綴;② B=01 是 D=011 的前綴。
例如收到 011…:可能是「B(01)+下一字以 1 開頭」也可能是「D(011)」→ 無法唯一解碼。
老師批改聚焦在 B⊂D(「電腦無法辨視 B 與 D」),但完整作答把 A⊂C 也寫出來更穩。

✓ 怎麼改成合法前綴碼

讓每個字元都落在二元樹的葉節點。例:A=0, B=10, C=110, D=111 → 互不為前綴,可唯一解碼。
畫樹自我檢查:沿著 0/1 往下走,只有葉子能放字元,路徑中途不能停字元。

Q3 · 哪一張是高通濾波(High-pass)結果?

✓ 你答對了:選 A(這題沒扣分,但把「為什麼」講滿才拿得到解釋分)。

A = 高通 High-pass

通過高頻=影像中劇烈變化的部分=邊緣、輪廓、細節、文字筆畫;壓掉平滑區 → 平坦處變黑,只剩輪廓線條。圖 A 看得到字母與圖形的邊緣 → 高通。

B = 低通 Low-pass

通過低頻=平滑、整體亮度、緩慢漸層;壓掉高頻細節 → 影像變模糊。圖 B 朦朧模糊 → 低通。
💡 完整解釋句:「A 是高通結果,因為高通濾波保留高頻分量(邊緣/細節),衰減低頻(平滑區),所以影像只剩物體輪廓。」
這跟 DCT 的「右下=高頻=細節、左上=低頻=平滑」是同一個頻率域概念,可以一起記。

✗ 你的錯誤(這題只拿一半)

你建的是歪斜樹:把 E 直接放在樹根當葉子 → 給了 E=0(只有 1 bit)。
為什麼錯:1-bit 碼字代表「E 在樹根的其中一邊,另一邊是其他全部」。但要這樣,E 的機率必須 ≥ 另一邊(≈79%)。E 只有 21%,根本不夠 → E 不可能是 1 bit。正確下 E 應該是 2 bits(01)。
根因:你沒有「每一步重新挑兩個最小」,而是把字元一個一個往下掛成一條長鏈。只要嚴格照合併表做,就不會歪。
💡 驗算技巧:① 機率大的字元碼一定要短(E、A 最短);② 把每個碼字長度 × 機率加總,如果比別人的樹大很多,代表你的樹不是最佳。

🎯 期末 8 大重點

1 · 亮度(Luminance）vs 色度(Chrominance)

亮度 Y色度 Cb, Cr
攜帶明暗 / 灰階細節顏色資訊(色差)
人眼非常敏感較不敏感
定義Y=0.299R+0.587G+0.114BCb=B−Y, Cr=R−Y(縮放)
壓縮保留完整解析度可次取樣(4:2:2 / 4:2:0)
💡 考點:為何先 RGB→YCbCr?因為人眼對亮度敏感、對色度不敏感,把色度資料丟一些幾乎看不出來,可省位元。

2 · 彩色影像壓縮流程(JPEG)

RGB 影像 │ ① 色彩轉換 RGB → YCbCr │ ② 色度次取樣(4:2:0,Cb/Cr 解析度減半) │ ③ 每個通道切成 8×8 區塊 │ ④ 對每塊做 DCT(空間域 → 頻率域) │ ⑤ 量化 Quantization(除以量化表並四捨五入) ← 失真就在這 │ ⑥ Zig-zag 掃描 → 行程長度編碼 RLE │ ⑦ 熵編碼 Huffman / 算術編碼(無失真) │ 壓縮檔 .jpg

解壓縮 = 反向:熵解碼 → 反量化 → IDCT → 色度上取樣 → YCbCr→RGB。

💡 唯一造成失真(lossy)的步驟是⑤ 量化;⑥⑦ 是無失真。

3 · DCT 係數的解讀

對 8×8 區塊做 2D-DCT,得到 8×8 的係數矩陣:

  • DC 係數(左上角 [0,0]):整塊的平均亮度,數值最大、最重要。
  • AC 係數(其餘):代表各種空間頻率。
    • 靠左上 = 低頻 = 平滑、緩慢變化(能量集中於此)。
    • 靠右下 = 高頻 = 細節、邊緣、雜訊。
  • 能量集中(energy compaction):自然影像大部分能量集中在少數低頻係數,高頻接近 0。
💡 這正是量化能壓縮的原因:把人眼不敏感的高頻係數丟掉/歸零,檔案變小而看不太出來。

4 · 卷積層(Convolutional Layer)

  • 用一組可學習的小濾波器 / 卷積核(kernel,如 3×3)滑過影像。
  • 每個位置算內積(加權和) → 產生特徵圖(feature map)。
  • 用途:偵測局部特徵 — 淺層找邊緣/角點/紋理,深層組成形狀/物件。
  • 兩大特性:
    • 局部連接:每個神經元只看小範圍(receptive field)。
    • 權重共享:同一核掃整張圖 → 參數大幅減少、具平移不變性。
  • 超參數:濾波器數量(深度)、核大小、stride 步幅、padding 補邊。後接非線性激活(ReLU)。

5 · 池化層(Pooling Layer)

  • 對特徵圖降取樣(down-sampling),縮小寬×高。
  • 在小視窗(如 2×2)上取值:最大池化 Max(取最大,最常用)、平均池化 Average(取平均)。
  • 好處:減少參數與計算、提供平移不變性、抑制過擬合、擴大感受野。
💡 常考是非題:池化層沒有可學習參數(它是固定運算,不像卷積有權重)。

6 · CNN 訓練的兩個問題

① 梯度消失 Vanishing Gradient

深層網路反向傳播時,梯度經過多層連乘變得極小 → 接近 0,前面幾層權重幾乎不更新、訓練很慢。
解法:ReLU 激活、適當權重初始化(He/Xavier)、Batch Normalization、殘差(skip)連接。

② 過擬合 Overfitting

模型把訓練資料細節甚至雜訊都背下來,在沒看過的資料上表現差(訓練準、驗證爛)。
解法:Dropout、資料擴增、L2 正則化、Early Stopping、更多資料。

(課程投影片 "Two Training Issues of CNN" 明列的就是這兩個。)

7 · 三個因子的兩難(Trade-off)

壓縮的三難:壓縮率、影像品質、運算成本/速度,三者無法兼得。

壓縮率(檔案小) /\ / \ 影像品質 —— 運算速度/成本

數位浮水印的三難(經典題)三個互相牽制的因子:

  • 強韌性 Robustness — 能撐過壓縮/雜訊/裁切等攻擊。
  • 不可察覺性 Imperceptibility — 看不出來,不傷畫質。
  • 容量 Capacity — 可嵌入的資料量。
💡 提高容量或強韌性,通常會犧牲不可察覺性 → 三者需取平衡。

8 · 浮水印:LSB vs DCT 兩種做法

LSB(最低有效位元)DCT(頻率域)
domain空間域頻率(轉換)域
做法把像素值的最低位元換成浮水印 bit做 DCT,改中頻係數嵌入,再做反 DCT
容量高較低
強韌性弱(壓縮/濾波就毀了)強(能撐過 JPEG 壓縮)
複雜度簡單、快較複雜、慢
用途易碎浮水印、簡單隱寫強韌/版權浮水印
💡 為何 DCT 嵌「中頻」? 低頻一改會被看見(傷畫質);高頻會被壓縮丟掉(不強韌)→ 中頻兼顧不可察覺與強韌。
